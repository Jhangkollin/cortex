# SP-VOICE — Real, Style-Configurable Brand-Voice Preview for Onboarding Step 5 (Design)

**Date:** 2026-05-20 · **Status:** approved (brainstorm), pending spec review · **Branch:** `feat/sp-voice` (worktree `cortex-wt/sp-voice`, off `develop` @ `761e001` — #36 SP-MEDIA + #37 user-flow merged; #38 SP-QUESTIONS NOT merged) · **Next:** writing-plans → subagent-driven

## 1. Problem

Onboarding **step 5 `StepLaunch`** ("Launch settings + brand voice preview") shows a brand-voice preview whose per-style `sample` copy is the **static `VOICE_TONES` mock constant** (3 hardcoded English answers). `StepLaunch` reads `VOICE_TONES` directly in both EN and 繁中; there is **no `OnboardingApi` seam method, no projection, no `page.tsx` wiring** for voice. The previewed voice is fictional and brand-agnostic.

## 2. Goal & Non-Goals

**Goal:** Step 5's brand-voice preview is **real** — each of the 3 fixed styles' `sample` is LLM-generated, grounded in the brand's **real** profile (incl. the real SP-3a-extracted `voice_samples`), persisted and deterministic per brand, switchable instantly — EN **and** 繁中.

**Non-goals (v1):** Databricks/snapshot (there is none); LLM-generated style `label`/`desc`/`icon` (presentation stays client-side constants); a configurable/extensible style registry; cross-feature grounding (SP-QUESTIONS/SP-MEDIA reuse); the rule-of-three spine extraction (deferred — §7).

## 3. Decisions (decision log)

## 4. Architecture (clone of `service/media_network/`, MINUS Databricks)

### §4.1 Domain `service/voice/`
- Migration (Alembic): **one** table `brand_voice` — `brand_id` UUID PK FK→`brand.id` (CASCADE), `status` ENUM `voicejobstatus` {pending,running,succeeded,failed}, `error` TEXT NULL, `samples` JSONB `server_default '{}'::jsonb` (shape `{expert,warm,playful}`), `created_at`/`updated_at` (`server_default sa.func.now()` + `onupdate` per CLAUDE.md), explicit `sa.Enum(name='voicejobstatus').drop(...)` in `downgrade()`. **`down_revision` = the real develop@761e001 alembic head** (verify at plan time via `alembic heads`; appears to be `f6a7b8c9d0e1_brand_onboarded_at` from #37). Pick a **fresh, distinct** revision id (NOT `f6a7b8c9d0e1` — see §7 collision note). No `*_member`/`weekly_*`-analog snapshot table.
- `service/voice/config.py` — `Config(BaseSettings, env_prefix="CORTEX_VOICE_")`: `stale_job_seconds:int=180` (no `dbx_catalog`, no `count`).
- `service/voice/model/job.py` ONLY (no snapshot model — there is no Databricks half): `VoiceJobStatus` StrEnum + `BrandVoice(SQLModel, table=True)` (`brand_id` PK FK→`brand.id`, `status` named-enum `sa_column`, `error`, `samples: dict[str,str]` JSONB default `{}`, timestamps) — structural twin of `media_network/model/job.py` with `outlets`→`samples` (a dict, not a list) and no `member.py`-analog.
- `service/voice/repo/brand_voice_repo.py` — one repo, twin of `brand_media_repo.py`: `get`/`find_in_flight`(PENDING/RUNNING)/`create`(insert-or-reset; **`samples` reset = Python `{}` NOT the string `"{}"`** — the e91ac41-class JSONB fix)/`mark_running`/`mark_succeeded(samples)`/`mark_failed`/`sweep_stale`. No `member_repo`/`question_repo`-analog.
- `service/voice/generator.py` (matcher-analog): `async generate_voice(profile: dict, provider, styles=("expert","warm","playful")) -> dict[str,str]`. LLM `complete_json`, schema `{"samples":{"expert":str,"warm":str,"playful":str}}`. System prompt: write one representative brand answer in EACH named style, grounded in the real profile (`name/about/tagline/category/products/competitors`) and the brand's real `voice_samples`; invent no facts. Server: keep only keys ⊆ `styles`; for any missing/empty/non-str → deterministic grounded fallback (`_fallback(style, profile)` built from the brand's own `voice_samples`/`about`, never empty); broad `except Exception  # noqa: BLE001` → all-fallback; per-item guards; return exactly `{style: str for style in styles}`. Never raises. Uses the same `CORTEX_ANALYZE_*` provider as media/questions (`build_provider(AnalyzeConfig)`).
- `service/voice/job_service.py` — `VoiceJobService`, structural twin of `MediaNetworkJobService` **minus the snapshot repo**: ctor `(database_client, brand_voice_repo, config, brand_profile_repo: BrandProfileRepo, provider=None, _generate=generate_voice)`. `_tasks`/dedupe/`start(brand_id, regenerate=False)`/`get`(raises the sibling's `NotFoundError`)/`sweep_stale`/`drain`/`cancel_all`(fail-mark-before-cancel). `_run`: `mark_running` → `profile_row = await self._brand_profile_repo.get(session, brand_id)` (NOT raw SQL) → build profile dict incl. `voice_samples`/`about`/`tagline`/`category`(←industry_vertical)/`products`/`competitors`/`name` → `await self._generate(profile, self._provider)` → `mark_succeeded(samples)`; `except asyncio.CancelledError: raise` BEFORE `except Exception` → `mark_failed`.
- `service/voice/container.py` — twin of `media_network/container.py` MINUS `databricks_client`/snapshot/`run_snapshot_sync`: `providers.Container(InfraContainer)`, `config`/`database_client`/`brand_voice_repo`/`brand_profile_repo`/`analyze_config`/`provider` Singletons, `job_service = Singleton(VoiceJobService, …)`.
- `app/api/voice/{dto.py,router.py}` — `BrandVoiceResponse{brand_id:UUID,status:str,samples:dict[str,str],error:str|None}` (`from_model` uses `m.status.value`). `POST /v1/brand/{brand_id}/brand-voice` (202, `?regenerate`) gated `EDIT_BRAND_SETTINGS`; `GET` same gated `VIEW_BRAND_DASHBOARD`; `active_brand`+`@inject Provide[VoiceContainer.job_service]`; `svc.start(tenant.brand_id, …)`/`svc.get(tenant.brand_id)`. `main.py` 6-edit wiring + lifespan `sweep_stale`/`cancel_all` mirroring media. **No `scripts/` sync entrypoint, no seed.**

### §4.2 Web (d8345ff-correct)
- `OnboardingApi`: add `getVoiceTones(): Promise<VoiceTone[]>`. `MockOnboardingApi`→ existing `VOICE_TONES` constant unchanged. `HttpOnboardingApi`→ Server Actions `startBrandVoiceAction`/`pollBrandVoiceAction` → endpoint; poll loop mirrors `getMediaNetwork` (bounded, rethrow on failed/timeout).
- `cortex-api.ts`: `startBrandVoice`/`pollBrandVoice` + canonical `BrandVoiceDTO` (`{brand_id,status,samples:Record<string,string>,error?}`). `projection.ts` `import type`s it; `projectVoiceTones(dto): VoiceTone[]` = the 3 client-side `VOICE_TONES` constants with `sample` replaced by `dto.samples[id]` **and falling back to the constant `sample` when a style is absent/empty** (null-safe). `id/label/desc/icon` always from the constant.
- Both `onboarding/v2/page.tsx` (EN) **and** `onboarding/v2/zh-TW/page.tsx`: add `voiceTones` state; fetch in `runAnalyze()` post-analyze in its **OWN isolated `void (async()=>{…})()` block separate from the media AND questions blocks**; reset in `restart()`; pass `voiceTones={voiceTones}` to `<StepLaunch/>`. Make both `StepLaunch` consume a `voiceTones: VoiceTone[]` prop instead of importing `VOICE_TONES` (keep the existing `voiceTone`/`setVoiceTone` selected-id state + instant client-side hot-swap; 繁中 copy preserved). Note: `StepLaunch` also takes a `mediaNetwork` prop from SP-MEDIA — that wiring stays untouched.
- Extend the structural call-site guard (sibling of `orchestrator-media-callsite.test.ts`): assert for BOTH EN+繁中 `page.tsx` that `getVoiceTones` ∉ `loadModeled`, ∈ `runAnalyze`, after `analyzeBrand`.

### §4.3 Error handling
Generator never raises (degrade → grounded per-style fallback from `voice_samples`/`about`; nothing invented). Web `getVoiceTones` failure non-fatal and **independent** of the media+questions fetches (own isolated catch). Preview never empty (constant `sample` fallback in projection). Job failure-isolated; `BrandProfileRepo` missing profile → fallback-only generation (still real-ish, never crashes).

### §4.4 Testing
Mirror the SP-MEDIA/QUESTIONS suite: generator unit (⊆-3-styles, missing/extra-key, never-raise on malformed LLM ×shapes, deterministic grounded fallback, grounding includes `voice_samples`); `brand_voice_repo` integration (idempotent, **create-resets-stale-row `samples`==`{}` as a dict not `"{}"`**); job integration (dedupe/sweep, `BrandProfileRepo` not raw SQL, `grep` no cross-domain SQL); API integration (flow→succeeded; **POST-403 without `EDIT_BRAND_SETTINGS`**; cross-tenant 4xx); web (projection unit, `StepLaunch` behavioral renders real prop / not the constant in EN+繁中, structural call-site guard both orchestrators); gate parity (`api make lint`/`make test`, alembic round-trip incl. `voicejobstatus` enum-drop, web type-check/lint/test). Re-review via **`@owl review`** (`@owl verify` is infra-broken).

## 5. Security
No Databricks → no Databricks creds anywhere. LLM via the existing `CORTEX_ANALYZE_*` provider (no new secret). No credentials in code. Tenant scoping via `active_brand` (cross-tenant → 4xx); service called with `tenant.brand_id`, not the URL path param. Generated samples are brand-marketing copy (no PII).

## 6. Scope
SP-VOICE (step 5) only — the **final** sub-project of the brand-onboarding enhancement (after SP-MEDIA #36 merged, SP-QUESTIONS #38 open).

## 7. Deferred / risks (recorded)
1. **Rule-of-three extraction (the deferred §7 decision):** with the 3rd consumer (voice) snapshot-less, the genuinely-shared surface is the **async-job + per-brand-result + status-enum + DI + endpoint + d8345ff-web** scaffolding (snapshot-*optional*), NOT the Databricks half. Extraction is justified but is a **separate, independently-reviewed post-SP-VOICE refactor PR** retrofitting all three (media/questions/voice) — explicitly NOT bundled into SP-VOICE (ship the final feature on the proven pattern; refactor under its own risk budget).
2. **Alembic revision-id collision (flag for #38, not SP-VOICE's bug):** develop@761e001 (#37) added a migration with revision id **`f6a7b8c9d0e1`** (`brand_onboarded_at`); the un-merged SP-QUESTIONS branch independently used revision id **`f6a7b8c9d0e1`** for `weekly_questions`. When #38 merges to develop these collide — #38 must rebase/rename its revision at merge. SP-VOICE deliberately picks a **distinct fresh** revision id with `down_revision` = the real develop@761e001 head (verified at plan time).
3. Deferred: configurable/extensible style registry; LLM-generated style label/desc; cross-feature grounding (answer a real SP-QUESTIONS question in each voice) — out of v1 scope.
